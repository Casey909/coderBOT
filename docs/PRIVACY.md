# Privacy and External Services Documentation

## Overview

This document details all external services and network communications used by CoderBOT, ensuring transparency about data handling and privacy.

## Core Principle

**CoderBOT operates primarily locally** with minimal external communication required only for:
1. Telegram Bot API (required for bot operation)
2. Optional AI transcription services (user-configurable)
3. XTerm.js CDN resources (for terminal rendering)

## Required External Services

### 1. Telegram Bot API (REQUIRED)

**Endpoint**: `https://api.telegram.org/*`

**Purpose**: 
- Receive user commands and messages
- Send bot responses
- Download user-uploaded files
- Send screenshots and media

**Data Sent**:
- Bot responses and terminal screenshots
- Confirmation messages
- Media files generated by bot

**Data Received**:
- User commands and messages
- User-uploaded files (photos, videos, audio)
- File metadata

**Privacy Notes**:
- This is the core communication channel for the bot
- All data passes through Telegram's servers
- Cannot be disabled (bot would not function)
- Subject to Telegram's privacy policy

**Configuration**:
```env
TELEGRAM_BOT_TOKENS=your_bot_token_here
```

**Security Measures**:
- ✅ URL validation implemented for all Telegram API calls
- ✅ File downloads are validated and sanitized
- ✅ Bot tokens are never logged or exposed
- ✅ Access control via `ALLOWED_USER_IDS`

## Optional External Services

### 2. OpenAI Whisper API (OPTIONAL - Audio Transcription)

**Endpoint**: OpenAI API endpoints

**Purpose**: Convert voice/audio messages to text

**Data Sent**:
- Audio files uploaded by users

**Data Received**:
- Transcribed text

**Privacy Notes**:
- **DISABLED BY DEFAULT** - Only active if `TTS_API_KEY` is configured
- Audio files are sent to OpenAI for processing
- Subject to OpenAI's privacy policy and data retention
- Temporary audio files are deleted after transcription

**Configuration**:
```env
# Leave empty to disable audio transcription
TTS_API_KEY=sk-proj-your_openai_key_here
```

**When to Use**:
- ✅ If you want voice-to-text functionality
- ✅ If you accept OpenAI's data handling practices
- ❌ If you require complete privacy (leave disabled)

**Security Measures**:
- Feature can be completely disabled by not setting API key
- Local audio files are cleaned up after use
- No persistent storage of audio data

### 3. Google Gemini API (OPTIONAL - Audio Transcription)

**Endpoint**: Google Gemini API endpoints

**Purpose**: Convert voice/audio messages to text (alternative to OpenAI)

**Data Sent**:
- Audio files uploaded by users

**Data Received**:
- Transcribed text

**Privacy Notes**:
- **DISABLED BY DEFAULT** - Only active if `TTS_API_KEY` is configured with Gemini key
- Audio files are sent to Google for processing
- Subject to Google's privacy policy and data retention
- Temporary audio files are deleted after transcription

**Configuration**:
```env
# Leave empty to disable audio transcription
TTS_API_KEY=AIza...your_gemini_key_here
```

**When to Use**:
- ✅ If you prefer Google's AI services over OpenAI
- ✅ If you accept Google's data handling practices
- ❌ If you require complete privacy (leave disabled)

## CDN Resources

### 4. jsDelivr CDN (XTerm.js Library)

**Endpoint**: `https://cdn.jsdelivr.net/npm/xterm@5.3.0/*`

**Purpose**: Load XTerm.js terminal emulator for rendering terminal screenshots

**Resources Loaded**:
- `xterm.css` - Terminal styling
- `xterm.js` - Terminal rendering library

**Privacy Notes**:
- Resources are loaded in an isolated Puppeteer headless browser
- No user data is sent to the CDN
- Only loaded during terminal screenshot generation
- Puppeteer runs with `--disable-background-networking` flag

**Security Measures**:
- ✅ CDN URL validation implemented
- ✅ Only trusted CDN domains allowed
- ✅ Resources loaded in isolated browser context
- ⚠️ Consider bundling locally for fully offline operation

**Alternative**:
For fully offline operation, XTerm.js could be bundled locally instead of loaded from CDN.

## Local-Only Features

The following features operate **completely locally** with no external communication:

### ✅ Terminal Sessions
- PTY (pseudo-terminal) sessions via `node-pty`
- All terminal I/O happens locally
- No data leaves your machine

### ✅ File Management
- All files stored in configured local directories
- No cloud storage or external backup
- Media watcher operates on local filesystem

### ✅ Session State
- All session state stored in memory
- No external state management
- No cloud sync

### ✅ User Authentication
- Environment variable based (`ALLOWED_USER_IDS`)
- No external authentication services
- Access control fully local

### ✅ Command Processing
- All command parsing and routing is local
- No external command validation

## Data Storage

### Local Storage Locations

All data is stored locally in the following directories:

```
MEDIA_TMP_LOCATION=/tmp/coderBOT_media (configurable)
├── bot-1/
│   ├── received/    # User-uploaded files
│   ├── audio/       # Temporary audio files (deleted after use)
│   └── ...          # Media to send to users
├── bot-2/
│   └── ...
```

**Storage Retention**:
- Terminal history: Kept in memory until session closes
- Media files: Persist until manually deleted or moved to 'sent' folder
- Audio files: **Deleted immediately after transcription**
- Session logs: Not persisted by default

## Privacy Best Practices

### For Maximum Privacy

1. **Disable Audio Transcription**
   ```env
   # Leave TTS_API_KEY empty
   TTS_API_KEY=
   ```

2. **Use Auto-Delete for Messages**
   ```env
   MESSAGE_DELETE_TIMEOUT=10000  # Delete confirmations after 10 seconds
   ```

3. **Clean Media Directory**
   ```env
   CLEAN_UP_MEDIADIR=true  # Clean on startup
   ```

4. **Restrict User Access**
   ```env
   ALLOWED_USER_IDS=your_telegram_id_only
   AUTO_KILL=true  # Shutdown on unauthorized access
   ```

### For Convenience (with AI Features)

1. **Enable Audio Transcription**
   ```env
   TTS_API_KEY=your_api_key_here
   AUDIO_TRANSCRIPTION_DEFAULT_MODE=copy  # Or 'prompt'
   ```

2. **Keep Media Files**
   ```env
   CLEAN_UP_MEDIADIR=false
   ```

## Compliance Notes

### GDPR Considerations

If operating in EU or with EU users:

1. **Telegram API**: User data passes through Telegram (subject to Telegram's GDPR compliance)
2. **AI Transcription**: Audio data sent to OpenAI/Google (check their GDPR compliance)
3. **Local Storage**: You are the data controller for locally stored files

2. Obtain consent for optional AI features
3. Implement data retention policies

## Questions and Concerns

### "Is my terminal session data sent anywhere?"

**No.** All terminal I/O is completely local via `node-pty`. The only thing sent to Telegram is:
- Screenshots of the terminal (as images)
- Text output (if in airplane mode)

### "Is my code/data safe?"

**Yes**, as long as you:
- Don't enable audio transcription (or accept the privacy implications)
- Trust Telegram with message content
- Keep your bot tokens secure

### "Can I run this completely offline?"

**Almost.** You need:
- Internet for Telegram API (bot can't work without it)
- Optionally bundle XTerm.js locally to eliminate CDN dependency

### "What happens to files I upload?"

1. Downloaded from Telegram API
2. Stored locally in `MEDIA_TMP_LOCATION/bot-N/received/`
3. Persisted until you delete them
4. Never sent anywhere else (unless you enable audio transcription)
